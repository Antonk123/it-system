15–20 minuter, live i mitt eget Prefabmastarna-system. Ingen påklistrad demo-data — det är hur jag använder systemet dagligen. Ni klickar runt själva i slutet om ni vill.

  1. Publika formuläret med AI som löser problem direkt

    Användare beskriver problem → AI söker er kunskapsbas → föreslår lösning på svenska → användare klickar "Det löste det" och inget ärende skapas. Eller "Behöver mer hjälp" och ärendet skapas som vanligt. Jag visar live med riktiga ärenden från min vardag.

  2. Ärendevyer — kanban, lista, dashboard

    Hur det ser ut när er IT-vardag faktiskt levs i systemet. Filter, anpassade vyer, sökning över allt med FTS5. Inget av detta är fluff — alla funktioner jag visar använder jag dagligen.

  3. AI-utkast på svar baserat på er KB

    När ärende ändå skapas: ett klick → AI utkastar svar baserat på era egna interna dokument. Granska, justera, spara som lösning. Aldrig auto-skickat.

  4. E-post in/ut med M365 OAuth2 + trådning

    Inkommande mejl blir ärenden. Svar på utgående mejl trådas tillbaka som kommentarer. OAuth2 mot Exchange Online — utan det räcker inte längre. Bekräftelsemejl med ärendenummer skickas automatiskt.

  5. Kunskapsbas, tidsrapportering, fakturering, push, webhooks

    Allt som faktiskt behövs i ett IT-system för en svensk SMB, utan plugin-marketplaces och separata moduler. Jag visar det ni vill se mest av — säg till.

För er som vill veta hur det är byggt

Tekniska faktum, inga buzzwords.

Om ni granskar produkter ni installerar — bra. Här är det som faktiskt är relevant.

Self-hosted

En Docker-instans på er Proxmox, Hetzner, eller vad ni nu kör. Inga API-anrop till oss. Backupar er data, inte vår.

Stack

React + TypeScript + Vite frontend. Node.js + Express backend. SQLite via better-sqlite3 med FTS5 för sök. Allt körs i en behållare.

AI-provider

Claude API (Anthropic) som default. Konfigurerbart via env. Tokens loggas per-installation för kostnadsuppföljning.

M365 OAuth2

XOAUTH2 via Azure MSAL för IMAP mot Exchange Online. Fallback till lösenord om ni inte vill gå OAuth-vägen.

Säkerhet

JWT + refresh-rotation, CSRF Double-Submit-Cookie, rate limits, helmet headers, bcrypt-hashade lösenord. Alla skrivningar via authenticerad användare.

GDPR & data

Er data ligger i er Docker-volym. AI-anrop går till Anthropic (EU-routing möjlig). Vi har ingen tillgång till er produktionsdata utan tillstånd.

Vanliga frågor från IT-admins

Det ni brukar fråga.

Kan jag testa själv innan demo?

Mejla mig så fixar jag inloggning till en sandbox-instans där ni kan klicka runt själva utan att jag står bakom axeln. Sandbox är read-only mot riktig data; ni kan skapa egna testärenden.

Vad händer om jag är missnöjd?

Säg upp avtalet. Ni får er data exporterad som SQLite-fil + JSON. Inga utträdes-avgifter, ingen lock-in. Er Docker-instans kan ni köra vidare själva eller stänga av.

Krävs det att vi har en M365-prenumeration?

Nej. Mejl-integrationen funkar mot vilken IMAP-server som helst. M365 OAuth2 är specifikt stödd eftersom det är den vanligaste setupen hos svenska SMB:s. Vill ni ha Gmail eller en egen mailserver så funkar det med standard IMAP-lösenord.

Vad händer om AI-leverantören går ner?

Alla AI-funktioner är designade non-blocking — kärnflödet (skapa, lösa, stänga ärenden, e-post in/ut) påverkas inte. AI-knapparna visas helt enkelt inte tills tjänsten är tillbaka.

Hur ofta uppdateras systemet?

Jag pushar uppdateringar veckovis till stabil branch. Ni får ett mejl när det finns en uppdatering och kan välja när ni vill köra `git pull && docker-compose up -d --build`. Eller låt mig göra det åt er — det ingår i månadsavgiften.
