## Color

All values use existing CSS variables from `src/index.css` (slate theme / `.theme-default`).

| Role | CSS Variable | HSL Value | Usage |
|------|-------------|-----------|-------|
| Dominant (60%) | `--background` | 222 47% 6% | Page background, FAB popover backdrop |
| Secondary (30%) | `--card` / `--secondary` | 220 40% 9% / 220 35% 13% | FAB popover card surface, action bar |
| Accent (10%) | `--primary` | 214 100% 60% | FAB button background, submit CTA, auth badge |
| Destructive | `--destructive` | 0 84% 60% | Not used in this phase |

Accent (`--primary`, electric blue 214 100% 60%) is reserved for:
1. FAB button fill
2. "Skicka in" (quick-create submit) button
3. "Inloggad som [namn]" auth badge border/text
4. Keyboard focus rings

The secondary accent (`--accent`, cyan 199 85% 58%) is NOT used in this phase to keep quick-capture visually distinct and immediately findable.

## Component Inventory

New components introduced in this phase:

### 1. QuickCaptureFAB

A persistent floating action button rendered at layout level (`App.tsx` or layout wrapper). Appears on all authenticated pages.

- **Shape:** Circular, 56px diameter
- **Background:** `bg-primary` (electric blue)
- **Icon:** `Plus` from lucide-react, 24px, `text-primary-foreground`
- **Position:** `fixed bottom-6 right-6 z-50`
- **Shadow:** `shadow-lg` (existing token from `tailwind.config.ts`)
- **Hover state:** `hover:scale-105 hover:shadow-xl` with `transition-transform duration-200`
- **Active state:** `active:scale-95`
- **Entry animation:** `animate-scale-in` (existing keyframe: opacity 0 → 1 + scale 0.95 → 1, 0.3s ease-out)
- **Tooltip:** "Snabbt ärende" shown on hover via shadcn `Tooltip`

### 2. QuickCapturePopover

Floating card that opens from the FAB. Uses shadcn `Popover` component.

- **Trigger:** FAB button
- **Alignment:** `align="end"` and `side="top"` so it opens upward from the FAB
- **Card surface:** `bg-card border border-border rounded-lg shadow-xl p-6`
- **Width:** `w-[320px]`
- **Contents:**
  - Heading: "Nytt ärende" — 16px, weight 600
  - Title `Input` with placeholder "Vad behöver du hjälp med?"
  - Submit `Button` variant `default` (primary blue): "Skicka in"
  - Cancel/close via clicking outside (Popover dismiss behavior)
- **Focus management:** Title input receives focus automatically on popover open (`autoFocus`)
- **Submit on Enter:** `onKeyDown` handler on input to submit on Enter key
- **Loading state:** Submit button shows `Loader2` spinner + "Sparar..." during API call
- **Post-submit behavior:** Popover closes, sonner toast fires with link to new ticket

### 3. InloggedBadge (public form auth detection)

An inline badge shown in `PublicTicketForm.tsx` when a valid auth token is detected.

- **Shape:** Pill badge, `bg-primary/15 text-primary border border-primary/30 rounded-full px-3 py-1`
- **Icon:** `User` from lucide-react, 14px, inline left of name
- **Text:** "Inloggad som [firstName]" — 13px, weight 500
- **Position:** Rendered above the form title, replacing the name/email fields section

### 4. Clone Button (ticket detail action bar)

Added to the existing action bar in `TicketDetail.tsx` alongside Edit/Delete/Share buttons.

- **Component:** shadcn `Button` variant `outline`, size `sm`
- **Icon:** `Copy` from lucide-react, 16px (already imported in `TicketDetail.tsx`)
- **Label:** "Klona"
- **Placement:** Between "Dela" (Share) and "Radera" (Delete) in the action bar
- **Behavior on click:** Navigates to `/tickets/new` with `location.state = { cloneData: { title, description, category_id, priority, template_id, template_field_values } }`

## Interaction States

### FAB popover — title input

| State | Appearance |
|-------|-----------|
| Default | `border-input bg-input` |
| Focus | `ring-2 ring-ring ring-offset-2` (existing shadcn default) |
| Filled | Normal — no special styling |
| Empty + Submit attempted | `border-destructive` + inline error message below input |

### Quick-create submit button

| State | Appearance |
|-------|-----------|
| Default | `bg-primary text-primary-foreground` |
| Hover | `hover:bg-primary/90` |
| Loading | Disabled + `Loader2` spinner + "Sparar..." label |
| Disabled (empty title) | `opacity-50 cursor-not-allowed` |

### Public form — logged-in state

| State | Appearance |
|-------|-----------|
| Auth detected | InloggedBadge shown, name/email fields hidden (`hidden` class, not removed from DOM for accessibility) |
| No auth | Normal form with name/email fields visible |

## Layout Notes

The FAB and QuickCapturePopover are positioned at the layout level. The recommended injection point is `App.tsx`, inside the `ProtectedRoute` wrapper, after the main `<Routes>` block:

```
<ProtectedRoute>
  <Routes>...</Routes>
  <QuickCaptureFAB />   ← add here
</ProtectedRoute>
```

This ensures the FAB is present across all authenticated pages without adding it to individual page components.
