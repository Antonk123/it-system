Create `server/src/routes/reports.ts` with a single `GET /summary` route behind `authenticate` middleware (per D-01, D-02, D-05).

The route accepts optional query params `year` (string) and `month` (string). Response shape per D-03:
```json
{
  "totals": { "open": N, "inProgress": N, "waiting": N, "resolved": N, "closed": N, "total": N },
  "byCategory": [{ "category": "string", "count": N }],
  "trend": [{ "month": "YYYY-MM", "created": N, "closed": N }],
  "avgResolutionDays": N,
  "agingTickets": N
}
```

SQL queries (all synchronous via `db.prepare().get()` / `.all()`):

1. **Totals** — `SUM(CASE WHEN status = 'open' THEN 1 ELSE 0 END)` etc. on `tickets` table with optional WHERE for year/month filter on `created_at`.

2. **byCategory** — `SELECT c.label as category, COUNT(t.id) as count FROM tickets t JOIN categories c ON t.category_id = c.id` with same WHERE, `GROUP BY t.category_id ORDER BY count DESC`.

3. **Trend** — Two separate queries merged in JS (per Pitfall 4 in RESEARCH.md):
   - Created per month: `SELECT strftime('%Y-%m', created_at) as month, COUNT(*) as created FROM tickets WHERE ... GROUP BY month`
   - Closed per month: `SELECT strftime('%Y-%m', closed_at) as month, COUNT(*) as closed FROM tickets WHERE closed_at IS NOT NULL AND ... GROUP BY month`
   - Merge via `Map<string, { month, created, closed }>`, sort by month ASC.
   - When `year` param is set, filter to that year. When no year, filter to `created_at >= date('now', '-12 months')` (per D-04).

4. **avgResolutionDays** — `SELECT AVG(CAST((julianday(closed_at) - julianday(created_at)) AS REAL)) as avgDays FROM tickets WHERE closed_at IS NOT NULL` with year/month filter. Default to 0 if null.

5. **agingTickets** — `SELECT COUNT(*) as count FROM tickets WHERE status = 'open' AND julianday('now') - julianday(created_at) > 7`. No year/month filter (aging is always current).

CRITICAL: Month param zero-padding — when month param arrives (e.g. `month=3`), convert to zero-padded: `String(parseInt(month)).padStart(2, '0')` before using in `strftime('%m', created_at) = ?` comparisons.

```typescript
import { useQuery } from '@tanstack/react-query';
import { api } from '@/lib/api';

export interface ReportsSummary {
  totals: {
    open: number;
    inProgress: number;
    waiting: number;
    resolved: number;
    closed: number;
    total: number;
  };
  byCategory: { category: string; count: number }[];
  trend: { month: string; created: number; closed: number }[];
  avgResolutionDays: number;
  agingTickets: number;
}

export const reportsSummaryKeys = {
  all: ['reports', 'summary'] as const,
  filtered: (year?: string, month?: string) =>
    [...reportsSummaryKeys.all, { year, month }] as const,
};

export const useReportsSummary = (year?: string, month?: string) => {
  const params = new URLSearchParams();
  if (year && year !== 'all') params.append('year', year);
  if (month && month !== 'all') params.append('month', month);
  const qs = params.toString() ? `?${params.toString()}` : '';

  return useQuery<ReportsSummary>({
    queryKey: reportsSummaryKeys.filtered(year, month),
    queryFn: () => api.request<ReportsSummary>(`/reports/summary${qs}`),
    staleTime: 5 * 60 * 1000,
    gcTime: 10 * 60 * 1000,
  });
};
```

Use `api.request<T>()` (the public method on ApiClient) — NOT `api.get()` which does not exist (per Pitfall 5 in RESEARCH.md).
