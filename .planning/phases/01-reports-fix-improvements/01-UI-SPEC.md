## Spacing Scale

Standard 8-point scale. All values pre-populated from Tailwind default config.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding |
| sm | 8px | Button internal padding, chart margins (`p-2`) |
| md | 16px | Card content padding (`p-4` = 16px), section gaps |
| lg | 24px | Tab content vertical padding, KPI card grid gap |
| xl | 32px | Page header bottom margin |
| 2xl | 48px | Major section breaks between chart groups |
| 3xl | 64px | Not used in this phase |

Exceptions:
- KPICard uses `p-4` (16px) inner padding — matches `CardContent` default.
- Chart height in Översikt tab: 200–300px pixel height (not spacing token) — required for print compatibility (see RPT-04).
- Category bar chart dynamic height: `Math.max(200, byCategory.length * 40)` — data-driven, not a spacing token.
- Touch targets on "Skriv ut" button: minimum 36px height via `size="sm"` on shadcn `Button`.

## Typography

All roles use Plus Jakarta Sans unless noted. Pre-populated from tailwind.config.ts and existing KPICard patterns.

Exactly 2 font weights are used across all roles: **400** (regular) and **700** (bold).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Chart axis tick labels, tooltip body text |
| Label | 12px | 400 | 1.4 | KPI card labels (`text-xs uppercase tracking-wider`) — visual differentiation via uppercase + tracking, not weight |
| Heading | 16px | 700 | 1.4 | Card titles (`CardTitle`), tab trigger labels |
| Display | 24px | 700 | 1.2 | KPI card metric values (`text-2xl font-bold`) |

Notes:
- KPI card metric values use `bg-gradient-to-r from-primary to-accent bg-clip-text text-transparent` — gradient text is an established pattern in this codebase; do not change.
- Chart labels: `fontSize: 12`, tick style via recharts `tick` prop — do not use Tailwind classes inside SVG.
- "Inga kategorier" empty state text: body role (14px/400) with `text-muted-foreground` color.

## Color

All values are CSS variable references resolved at runtime. The theme is dark by default (:root block).

Primary focal point: KPI metric values (Display role, 24px, gradient text using `--primary` to `--accent`) in the Översikt tab — these are the first elements the eye should land on when the page loads.

| Role | CSS Variable | Resolved (dark) | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | `hsl(222 47% 6%)` | Page background, main surface |
| Secondary (30%) | `--card` | `hsl(220 40% 9%)` | Chart cards, KPI cards, tooltip backgrounds |
| Accent (10%) | `--accent` | `hsl(199 85% 58%)` | Reserved — see list below |
| Destructive | `--destructive` | `hsl(0 84% 60%)` | No destructive actions in this phase |

Accent (`--accent`) is reserved exclusively for:
1. Gradient text on KPI metric values (`from-primary to-accent bg-clip-text`) — existing pattern, unchanged.
2. KPICard icon container gradient (`from-primary/20 to-accent/10`) — existing pattern, unchanged.
3. Tooltip header gradient (`from-primary/50 via-accent/50`) — existing pattern, unchanged.
4. No new accent usage is introduced in this phase.

Chart color assignments (from existing `COLORS` and `STATUS_COLORS` arrays — do not change):

| Series | CSS Variable | Purpose |
|--------|-------------|---------|
| `COLORS[0]` / primary | `hsl(var(--primary))` | Category bar 1, trend "Skapad" bars |
| `COLORS[1]` / chart-2 | `hsl(var(--chart-2))` | Category bar 2 |
| `COLORS[2]` / chart-3 | `hsl(var(--chart-3))` | Category bar 3 |
| `COLORS[3]` / chart-4 | `hsl(var(--chart-4))` | Category bar 4, trend "Stängd" line |
| `COLORS[4]` / chart-5 | `hsl(var(--chart-5))` | Category bar 5 |

Tooltip background: `hsl(var(--card))` with `border: 1px solid hsl(var(--border))` and `borderRadius: 8px` — matches existing chart tooltip style in RESEARCH.md.

Print override: `body { background: white !important; color: black !important; }` applies only inside `@media print`. Cards get `border: 1px solid #ccc !important`. This is a print-only exception; no impact on screen rendering.

### New Component Specs

**"Skriv ut" button:**
- Variant: `outline`, size: `sm`
- Icon: `<Printer className="h-4 w-4" />` (lucide-react `Printer`)
- Label: `Skriv ut` (hidden on mobile: `hidden sm:inline`)
- Class: `gap-2 print:hidden`
- Placement: Alongside existing CSV Export and Customize buttons in the Reports page header flex row.

**Category breakdown chart (Översikt tab):**
- Type: `BarChart` with `layout="vertical"` (horizontal bars)
- Data key: `count` (Bar), `category` (YAxis)
- Height: `Math.max(200, byCategory.length * 40)` pixels via `ResponsiveContainer`
- Left margin: 80px to accommodate category label width
- Bar fill: `COLORS[index % COLORS.length]` via `<Cell>` — cycles the existing 5-color array
- Bar radius: `[0, 4, 4, 0]` (right-side rounded)
- Bar name: `Ärenden`
- Placement: After the status pie chart section within the existing **Översikt** tab

**Trend overlay chart (Trend tab):**
- Type: `ComposedChart` (replaces existing single-series `BarChart`)
- Bar series: `dataKey="created"`, name `"Skapad"`, fill `hsl(var(--primary))`, radius `[4,4,0,0]`
- Line series: `dataKey="closed"`, name `"Stängd"`, stroke `hsl(var(--chart-4))`, `strokeWidth={2}`, `dot={{ r: 3 }}`
- Height: 300px fixed pixel height (required for print compatibility)
- Legend: enabled via `<Legend />`

**Loading skeleton (new `useReportsSummary` loading state):**
- Show 4 `<Skeleton className="h-24 w-full" />` blocks in a grid replacing the KPI card row while `isLoading === true`.
- Show `<Skeleton className="h-[300px] w-full" />` replacing each chart while loading.
- Use the existing `skeleton.tsx` shadcn component — no new implementation.

## Interaction Contract

### Tab behavior (print)
- Active tab content: visible in print.
- Inactive `TabsContent` (`[data-radix-tabs-content][data-state="inactive"]`): `display: none !important` in `@media print`.
- Tab list (`[role="tablist"]`): hidden in print via `@media print`.
- This preserves the existing Radix tab visibility logic — no JS override needed.

### Filter controls (print)
- Year/month `<Select>` dropdowns: hidden in print via `.reports-filter-bar { display: none !important; }` in `@media print`.
- Add `reports-filter-bar` class to the filter container div.

### Navigation chrome (print)
- `nav`, `header`, `aside`, `[data-sidebar]`, `[data-print-hide]`: hidden in print via `@media print`.

### Chart rendering (print)
- `.recharts-responsive-container { height: 300px !important; }` in `@media print`.
- This prevents SVG height collapse to 0 when `ResponsiveContainer` cannot measure its parent.

### Year/month filter wiring
- Existing `selectedYear` and `selectedMonth` state in Reports.tsx passes directly to `useReportsSummary(selectedYear, selectedMonth)` as query params.
- Remove all `useMemo` aggregations that ran on the raw `tickets` array.
- `availableYears` for the year dropdown: derived client-side from `summary.trend` — extract unique YYYY prefix from each `trend[n].month` string. If trend is empty, default to current year.

### Module visibility
- Category chart registers as `categoryChart` in `useReportsPreferences` with `visible: true` default.
- Respects the existing `ReportsCustomization` module toggle system — chart is hidden when `preferences.modules.categoryChart.visible === false`.

## Print CSS Contract

Applied in `src/index.css`, appended after `@layer utilities`. This is the sole `@media print` block.

```css
@media print {
  nav, header, aside,
  [data-sidebar], [data-print-hide] {
    display: none !important;
  }

  [data-radix-tabs-content][data-state="inactive"] {
    display: none !important;
  }

  [role="tablist"],
  .reports-filter-bar {
    display: none !important;
  }

  .recharts-responsive-container {
    height: 300px !important;
  }

  body {
    background: white !important;
    color: black !important;
  }

  .card, [data-card] {
    border: 1px solid #ccc !important;
    break-inside: avoid;
  }
}
```

The `print:hidden` Tailwind variant is applied directly to the "Skriv ut" button: `className="... print:hidden"` — so the button does not appear in the printed output.
