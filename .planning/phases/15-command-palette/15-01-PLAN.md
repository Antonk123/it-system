**File 2: src/hooks/useCommandPaletteSearch.ts**

Create a custom hook that encapsulates debounced backend search for both tickets and KB articles.

```typescript
import { useState, useEffect, useRef, useCallback } from 'react';
import { api } from '@/lib/api';

interface SearchResult {
  id: string;
  title: string;
  type: 'ticket' | 'kb';
  subtitle?: string; // status for tickets, article_type for KB
}

interface UseCommandPaletteSearchReturn {
  results: SearchResult[];
  isSearching: boolean;
  search: string;
  setSearch: (value: string) => void;
}
```

Implementation:
- Accept no arguments. Manage `search` state internally.
- When `search` changes (non-empty, trimmed), set `isSearching = true` and start a 250ms debounce timer (use `useRef<ReturnType<typeof setTimeout>>` pattern from GlobalSearch).
- After debounce fires, call `api.getTickets('?page=1&limit=6&status=all&search=' + encodeURIComponent(term))` AND `api.getKbArticles({ search: term })` in parallel via `Promise.all`.
- Map ticket results: `{ id: t.id, title: t.title, type: 'ticket', subtitle: t.status }`. Take first 5.
- Map KB results: `{ id: a.id, title: a.title, type: 'kb', subtitle: a.article_type }`. Take first 5.
- Concatenate tickets first, then KB articles into `results`.
- When `search` is empty, set `results = []` and `isSearching = false`.
- Clean up timeout on unmount.
- Return `{ results, isSearching, search, setSearch }`.

For ticket response handling, replicate the GlobalSearch pattern: `const rows = Array.isArray(response) ? response : response.data;` since getTickets can return either format.

      - Knowledge Base (/kb) with `BookOpen` icon
