## Spacing Scale

Declared values (multiples of 4 only). All values map to Tailwind utility classes.

| Token | px | Tailwind | Usage in this phase |
|-------|----|----------|---------------------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-text gaps inside aging rows and reminder rows |
| sm | 8px | `gap-2`, `p-2` | Padding inside compact row cells; badge inner padding |
| md | 16px | `gap-4`, `p-4` | Panel card internal padding (matches existing KPICard `p-4`) |
| lg | 24px | `gap-6`, `p-6` | Vertical gap between dashboard section groups (panels below KPI grid) |
| xl | 32px | `gap-8` | Not used in this phase |
| 2xl | 48px | — | Not used in this phase |
| 3xl | 64px | — | Not used in this phase |

Exceptions:
- Panel section `space-y-5` (20px) inherits from existing `Dashboard.tsx` outer `space-y-5` — do not change this value to maintain visual consistency with the existing KPI grid spacing.
- Skeleton row height: 40px (`h-10`) — not a spacing token, it is a component dimension matching expected row height.

## Typography

All sizes use `Plus Jakarta Sans`. Line heights are unitless ratios.

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Body | 14px | `text-sm` | 400 | 1.5 | Aging ticket title, reminder ticket title, requester name |
| Label | 12px | `text-xs` | 500 | 1.4 | Panel header label (e.g. "Åldrande ärenden"), age badge text, KPI "idag" sub-label |
| Heading (page) | 20px | `text-xl` | 700 | 1.2 | Dashboard page heading "Översikt" (existing — do not change) |
| Display (KPI value) | 24px | `text-2xl` | 700 | 1.0 | KPI card main numeric value (existing — do not change) |

Additional constraints:
- KPI sub-label ("idag" line): `text-xs font-medium text-muted-foreground` — renders below the `text-2xl` value, separated by `mt-0.5`.
- "Visa alla" link: `text-xs font-medium text-primary underline-offset-2 hover:underline` — right-aligned in panel header.
- Age badge ("12 dagar"): `text-xs font-semibold tabular-nums` — always right-aligned in aging row to prevent layout shift.
- Reminder time: `text-xs font-medium text-muted-foreground tabular-nums`.
- Priority badge text: `text-[10px] font-bold uppercase tracking-wider` — matches existing badge usage across the app.

## Color

The project uses CSS variables defined per-theme in `index.css`. All color references below use those variables. The default theme (dark) is Slate.

| Role | CSS Variable | Approx Dark Value | Usage in this phase |
|------|-------------|-------------------|---------------------|
| Dominant (60%) | `--background` | hsl(222 47% 6%) | Page background, outer layout |
| Secondary (30%) | `--card` | hsl(220 40% 9%) | AgingTicketsPanel card, RemindersPanel card |
| Accent (10%) | `--primary` | hsl(214 100% 60%) | KPI sub-label count highlight, "Visa alla" link, panel section icon |
| Muted text | `--muted-foreground` | hsl(215 18% 60%) | Secondary text: requester name, reminder time, panel descriptors |
| Destructive | `--destructive` | hsl(0 84% 60%) | Critical priority badge only — no destructive actions exist in this phase |

**Accent reserved for (explicit list):**
1. "Visa alla" link text in AgingTicketsPanel header
2. Panel header icon color (e.g. `Clock` icon in AgingTicketsPanel, `Bell` icon in RemindersPanel)
3. The numeric count in the KPI "idag" sub-label when count > 0 (e.g. "+3 idag")
4. Skeleton pulse uses `--muted` as base — NOT accent

**Priority badge color mapping** (pre-existing, do not change):
- `low` → `hsl(var(--priority-low))` (green)
- `medium` → `hsl(var(--priority-medium))` (amber)
- `high` → `hsl(var(--priority-high))` (orange)
- `critical` → `hsl(var(--priority-critical))` (red)

**Aging severity tint on rows** (Claude's discretion — decided here):
- Tickets aged 14+ days: left border `border-l-2 border-l-destructive/60` — a narrow left-edge warning signal, no fill.
- Tickets aged 7–13 days: left border `border-l-2 border-l-[hsl(var(--priority-high))]/50`.
- Tickets aged 0–6 days: no left border.
This tint is purely decorative and must not replace the age-in-days text.

### KPICard extension

Add optional `subLabel` prop of type `string | React.ReactNode`. When provided, render it below the `text-2xl` value block:

```tsx
{subLabel && (
  <p className="text-xs font-medium text-muted-foreground mt-0.5">{subLabel}</p>

```

When `useDashboardOverview` is loading, pass `<Skeleton className="h-3 w-16 mt-1" />` as the `subLabel` value. This keeps the KPICard interface clean — the consumer decides whether to show a string or a skeleton.

## Interaction Contract

### AgingTicketsPanel

| State | Behavior |
|-------|----------|
| Loading | 5 skeleton rows `<Skeleton className="h-10 w-full rounded-md" />` with `space-y-3` |
| Empty (no aging tickets) | Empty state: heading "Ingen eftersläpning" + body "Alla öppna ärenden är nyligen uppdaterade." No icon needed — keep it minimal. |
| Populated (≤ 5 tickets) | List of compact rows; no "Visa alla" shown |
| Populated (> 5 tickets) | List of top 5 rows + "Visa alla" link in panel header right slot; navigates to `/tickets?status=open` |
| Row hover | `hover:bg-muted/40 transition-colors duration-150` — subtle fill, no translation |
| Row click | `navigate('/tickets/:id')` — full ticket detail |

### RemindersPanel

| State | Behavior |
|-------|----------|
| Loading | 5 skeleton rows `<Skeleton className="h-10 w-full rounded-md" />` with `space-y-3` |
| Empty (no upcoming reminders) | Empty state: heading "Inga kommande påminnelser" + body "Lägg till påminnelser på ärenden för att se dem här." |
| Populated | Up to 5 rows ordered soonest-first; if more than 5 exist, show "Visa alla" (links to `/tickets?status=open` as interim destination) |
| Row hover | `hover:bg-muted/40 transition-colors duration-150` |
| Row click | `navigate('/tickets/:id')` — full ticket detail for parent ticket |

### KPI "idag" sub-label

| State | Behavior |
|-------|----------|
| Loading | `<Skeleton className="h-3 w-16 mt-1" />` |
| Zero count | Display "+0 idag" in `text-muted-foreground` (no accent color — zero is neutral) |
| Positive count | Display "+N idag" in `text-primary font-semibold` |

### Panel entrance animation

Both panels use `animate-fade-in` (already defined in `tailwind.config.ts`) with staggered delay:
- AgingTicketsPanel: `animationDelay="0ms"` (renders first)
- RemindersPanel: `animationDelay="100ms"`

Apply via `style={{ animationDelay }}` on the outer wrapper div, matching the KPICard pattern.

## Aging Row Layout

Each aging ticket row is a `flex items-center gap-3 px-1 py-2 rounded-md cursor-pointer` with the optional left-border severity tint applied on the row element itself.

```
[left-border tint] | [title — flex-1 truncate text-sm] | [age badge text-xs tabular-nums text-muted-foreground w-16 text-right] | [priority badge]
```

- Title: `text-sm font-medium text-foreground truncate flex-1` — truncates if long; full title in `title` attribute for native tooltip.
- Requester name: displayed as secondary line `text-xs text-muted-foreground` below title when space allows. On narrow layouts, omit requester — title + age + priority are the minimum viable row.
- Age: right-aligned, fixed width `w-16`, prevents layout shift as numbers change.

## Implementation Notes for Executor

1. **Panel insertion point in Dashboard.tsx:** Add panels below the secondary stats row (`grid grid-cols-1 sm:grid-cols-2 gap-4` block at line ~137). The outer `space-y-5` div handles vertical gap automatically.

2. **KPICard subLabel for "idag":** The sub-label value is computed in `Dashboard.tsx` from `dashboardOverview.data?.todayCounts`. While `isLoading` is true, pass `<Skeleton className="h-3 w-16 mt-1" />`. Once loaded, pass `"+N idag"` string. Apply this to three KPICards: Öppna ärenden (created_today), Lösta (resolved_today), Arkiverade (closed_today).

3. **useDashboardOverview staleTime:** 60 seconds. `refetchOnWindowFocus: true` (React Query default). This prevents stale midnight counts on long-open tabs.

4. **useUpcomingReminders limit:** Return up to 5 from the API (LIMIT 5 in SQL, not LIMIT 10 as suggested in RESEARCH.md — 5 matches the aging tickets cap and fits the panel without scroll). Show "Visa alla" if the returned array length equals 5 (treat 5 as "there may be more").

5. **Route placement:** `/dashboard-overview` and `/upcoming-reminders` both go ABOVE `router.get('/:id', ...)` in `server/src/routes/tickets.ts`.
