## Interaction Contract

### Create Form — Progressive Disclosure

**Always visible (Grundläggande):**
- Titel (Input)
- Mall (TemplateCombobox — inline, next to or below Titel field)
- Beskrivning (RichTextEditor)
- Kategori (CategoryCombobox — replaces current Radix Select)
- Beställare (UserCombobox — existing, no change)

**Collapsed by default (Detaljer section):**
- Trigger label: "Detaljer" with a ChevronDown icon and a muted count badge showing non-default values (e.g. "1 valt" when priority is not Medium)
- Contents: Prioritet (Select), Status (Select — edit mode only)
- Collapsed state: trigger shows `bg-card` background, 1px `border-border` border, `rounded-md`
- Expanded state: content slides in using `accordion-down` keyframe (0.2s ease-out, already defined in tailwind.config.ts)

**Collapsed by default (Bilagor & Checklista section):**
- Trigger label: "Bilagor & Checklista" with ChevronDown icon and muted count badge
- Contents: FileUpload, TicketChecklist
- Same expand/collapse animation as Detaljer

**Collapsed default state:** `open={false}`, trigger height 44px minimum for touch targets.

### Edit Form — Hidden Empty Fields

Fields with existing content: always visible, no change to layout.

**Empty optional fields replaced by add-buttons:**
- Empty `solution`: render `<Button variant="ghost" size="sm">+ Lösning</Button>` — clicking expands RichTextEditor inline with a fade-in animation (opacity 0→1, translateY 8px→0, duration 200ms)
- Empty `notes`: render `<Button variant="ghost" size="sm">+ Interna anteckningar</Button>` — same expand animation

**Add-button styling:**
- `variant="ghost"`, `size="sm"`, text color `text-muted-foreground`
- Left-aligned, icon `PlusCircle` (16px) before label
- On hover: text shifts to `text-foreground`, background `bg-muted/40`

### Template Combobox — Inline on Create Form

- **Trigger:** `<Button variant="outline">` full-width or constrained to ~240px, shows selected template name or "Geen mall" placeholder
- **Popover:** `w-[280px]`, search Input at top (no Command wrapper — mirrors UserCombobox pattern, not cmdk Command)
- **Selecting a template:** auto-fills Titel, Beskrivning, Prioritet, Kategori (current behavior preserved — CONTEXT.md D-05)
- **Clear template action:** text link "Rensa mall" in muted-foreground, clicking restores free-form mode and clears auto-filled fields
- **Remove:** Dialog modal (`templateDialogOpen` state) and "Skapa från mall" Button are deleted (CONTEXT.md D-06)

### Category Combobox — Replaces Radix Select

- Same Popover + Input search pattern as UserCombobox
- Popover width: `w-[260px]`
- Footer row (always last in list): "Ny kategori..." entry — clicking reveals an inline Input + "Lägg till" Button below the combobox (existing inline-creation UX preserved — CONTEXT.md specifics)
- Fixes Radix Select scroll-jump bug on 9+ categories (CONTEXT.md specifics)

### Searchable Combobox — Keyboard Interaction

Both CategoryCombobox and TemplateCombobox must support:
- `Tab` to open/close popover trigger
- Type-to-search immediately on popover open (focus lands on search Input)
- `Enter` to select highlighted item
- `Escape` to close without selection
- `ArrowUp` / `ArrowDown` to navigate filtered list
