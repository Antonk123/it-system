## Spacing Scale

Declared values (multiples of 4 — Tailwind default scale, no custom spacing override):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (p-1 / gap-1) | Icon gaps, inline badge padding |
| sm | 8px (p-2 / gap-2) | Compact element spacing, between list rows |
| md | 16px (p-4 / gap-4) | Default card padding, form field spacing |
| lg | 24px (p-6 / gap-6) | Section padding, panel interior |
| xl | 32px (p-8 / gap-8) | Layout gaps between panels |
| 2xl | 48px (p-12) | Major section breaks (page-level top padding) |
| 3xl | 64px (p-16) | Page-level outer spacing (desktop only) |

Exceptions:
- Bottom tab bar height: 56px (14 Tailwind units) — matching common mobile nav convention and keeping touch targets at minimum 44px
- QuickCaptureFAB bottom offset on mobile: `bottom-[72px]` (56px bar + 16px gap) to float clearly above the tab bar
- Touch targets on bottom tab items: minimum 44px height enforced via `min-h-[44px]`

## Color

All colors are CSS-variable-driven and theme-aware. Specified as the default (Slate) theme values; other themes (Midnight, Graphite, Stone, Daylight) follow the same role mapping with different HSL values.

| Role | CSS Variable | Approximate Slate Value | Usage |
|------|-------------|------------------------|-------|
| Dominant (60%) | `--background` | hsl(222 47% 6%) — deep navy | Page backgrounds, main canvas |
| Secondary (30%) | `--card` / `--secondary` | hsl(220 40% 9%) / hsl(220 35% 13%) | Cards, panels, sidebar, bottom tab bar surface |
| Accent (10%) | `--primary` | hsl(214 100% 60%) — electric blue | Reserved-for list below |
| Destructive | `--destructive` | hsl(0 84% 60%) — red | Destructive actions only |
| Muted | `--muted` | hsl(218 32% 15%) | Skeleton pulse background (`bg-muted`) |
| Muted foreground | `--muted-foreground` | hsl(215 18% 60%) | Secondary text, placeholders, empty state copy |

Accent (`--primary`) reserved exclusively for:
- Active state indicator on bottom tab bar (active tab icon + label)
- Active route highlight in sidebar nav
- Primary action buttons (Skapa ärende, save actions)
- Focus ring (`--ring`) on interactive elements
- KPI card trend positive arrows
- Link text on hover

Accent is NOT used for: general text, borders, backgrounds, skeleton shapes.

Bottom tab bar surface: use `bg-card border-t border-border` — secondary color at 30%, matching panel surfaces consistently.

## Animation Contract

All animations respect `prefers-reduced-motion`. When reduced motion is active, set `duration: 0` on all Framer Motion variants — follow the pattern in `AnimatedNumber.tsx`.

### Framer Motion Variants

**Page transition (route change):**
```
initial:  { opacity: 0, y: 8 }
animate:  { opacity: 1, y: 0, transition: { duration: 0.2, ease: "easeOut" } }
exit:     { opacity: 0, y: -8, transition: { duration: 0.15, ease: "easeIn" } }
```
Implementation: Wrap route outlet in `App.tsx` with `<AnimatePresence mode="wait">`. Key each page by `location.pathname`.

**Staggered list/card reveal (page load):**
```
container: { animate: { transition: { staggerChildren: 0.05 } } }
item:      initial: { opacity: 0, y: 12 }
           animate: { opacity: 1, y: 0, transition: { duration: 0.25, ease: "easeOut" } }
```
Apply to: ticket list rows/cards, KB article cards, Dashboard panels (upgrade existing `animationDelay` CSS to this variant).

**Skeleton → content crossfade:**
```
exit:    { opacity: 0, transition: { duration: 0.15 } }
initial: { opacity: 0 }
animate: { opacity: 1, transition: { duration: 0.2 } }
```
Use `AnimatePresence mode="wait"` keyed on `isLoading`.

**Dashboard KPI entrance (upgrade from CSS animationDelay):**
Same stagger container pattern as list reveal above. Stagger delay: 0.07s per card. Cards 1→4 enter in sequence.

### Duration Budget
| Moment | Duration |
|--------|----------|
| Page fade out | 150ms |
| Page fade in | 200ms |
| List item reveal | 250ms |
| Stagger interval | 50ms |
| Skeleton crossfade | 150ms → 200ms |
| CSS micro-interactions (hover, focus) | 150ms via `transition-colors duration-150` |

## Bottom Tab Bar Design Contract

- **Height:** 56px (`h-14`)
- **Position:** fixed bottom-0 left-0 right-0, z-50
- **Surface:** `bg-card border-t border-border` — matches panel surface, integrates with theme
- **Items:** 4 — Dashboard (LayoutDashboard icon), Tickets (Ticket icon), KB (BookOpen icon), Settings (Settings icon)
- **Active state:** icon + label rendered in `text-primary` (accent color). Active indicator: 2px top border in `bg-primary` above the active tab item (pill/bar at top edge of the item, not below)
- **Inactive state:** icon + label in `text-muted-foreground`
- **Touch target:** each item is `flex-1 min-h-[44px] flex flex-col items-center justify-center gap-0.5`
- **Label:** `text-xs font-bold` — visible always (not hidden), 10px label below icon
- **Icon size:** 20px (w-5 h-5)
- **Safe area:** add `pb-safe` or `padding-bottom: env(safe-area-inset-bottom)` for iOS home indicator clearance

## Mobile Ticket Card Design Contract

Each ticket row on mobile renders as a card (`bg-card rounded-lg border border-border p-3`).

Fields shown (in order):
1. Title — `text-sm font-bold text-foreground truncate` (single line, truncated)
2. Status badge — existing shadcn Badge variant, on the same row as title right-aligned
3. Priority indicator — colored dot + `text-xs font-bold text-muted-foreground` label
4. Age — `text-xs text-muted-foreground` (e.g. "3 dagar sedan")

Tap target: entire card is tappable, navigates to ticket detail. Use `cursor-pointer` and `hover:bg-accent/5 active:bg-accent/10 transition-colors`.

Spacing between cards: `gap-2` (`8px`).
