  <action>
Rewrite `src/components/KBLinksSection.tsx` entirely. Keep the same export name and props interface (`{ ticketId, ticketTitle }`). Replace all manual useState/useEffect data fetching with React Query.

**Data fetching with React Query:**

1. `useQuery` for linked articles:
   - queryKey: `['ticket-kb-links', ticketId]`
   - queryFn: `() => api.getTicketKbLinks(ticketId)`
   - This runs on mount, shows linked articles immediately (KBSB-03)

2. `useQuery` for search results:
   - queryKey: `['kb-search', debouncedSearch]`
   - queryFn: `() => api.getKbArticles({ search: debouncedSearch })`
   - `enabled: debouncedSearch.length >= 2` (only search when 2+ chars typed)
   - Filter out already-linked article IDs from results client-side

3. `useMutation` for linking:
   - mutationFn: `(articleId: string) => api.linkKbArticleToTicket(ticketId, articleId)`
   - onSuccess: invalidate `['ticket-kb-links', ticketId]`, clear search, show `toast.success('KB-artikel lankad')`

4. `useMutation` for unlinking:
   - mutationFn: `(articleId: string) => api.unlinkKbArticleFromTicket(ticketId, articleId)`
   - onSuccess: invalidate `['ticket-kb-links', ticketId]`, show `toast.success('Lank borttagen')`

**Debounce search input:**
Use a `useRef<ReturnType<typeof setTimeout>>` + `useState` for `debouncedSearch`. On `searchQuery` change via useEffect, clear existing timeout, set new 300ms timeout that updates `debouncedSearch`. Clean up on unmount. Same pattern as `useCommandPaletteSearch.ts`.

**Layout structure (top to bottom):**

1. **Header row:** `BookOpen` icon + "Knowledge Base ({linkedCount})" label on the left. "Skapa KB-artikel" button on the right (same navigate logic as current).

2. **Linked articles list** (KBSB-03 — shown BEFORE search, always visible):
   - Show immediately from `useQuery` data
   - Each linked article: `BookOpen` icon, title as `<Link to={/kb/${id}}>`, category `<Badge>` with color styling, `X` unlink button on hover (`opacity-0 group-hover:opacity-100`)
   - Loading state: `<Loader2 className="w-4 h-4 animate-spin">` centered
   - Empty state: "Inga KB-artiklar lankade" in `text-xs text-muted-foreground`

3. **Search input** (KBSB-01):
   - Use shadcn `Input` component (not Command — replace the Popover/Command pattern with inline search)
   - `placeholder="Sok KB-artiklar..."` with `Search` icon from lucide-react as a left icon (position the icon absolutely inside a relative wrapper div, add `pl-8` to the Input)
   - `value={searchQuery}` controlled input
   - Show below the linked articles list, separated by `pt-3 border-t border-dashed`

4. **Search results** (KBSB-01 + KBSB-02):
   - Only render when `debouncedSearch.length >= 2`
   - Show a `Loader2` spinner while `isFetching` is true
   - Each result card: `p-2 rounded-lg border-l-2 border-primary/30 bg-muted/30 hover:bg-muted transition-colors` container. Inside: article title (text-sm font-medium), category badge with color, and FTS5 snippet. Render snippet with `dangerouslySetInnerHTML={{ __html: article.snippet }}` wrapped in `<p className="text-xs text-muted-foreground mt-0.5 line-clamp-2">` (FTS5 returns safe `<mark>` tags from our own server — single-user system, no XSS risk). Each result has a `<Button variant="ghost" size="sm">` with `Link2` icon to trigger linking (KBSB-02).
   - No results state: "Inga artiklar matchade sokningen" when search returned empty array and not fetching
   - Limit display to 8 results max (`.slice(0, 8)`)
