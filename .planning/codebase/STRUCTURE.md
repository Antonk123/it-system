## Directory Purposes

**`src/`:**
- Purpose: Frontend React application source code
- Contains: Components, pages, utilities, hooks, context, types
- Key files: `main.tsx` (entry), `App.tsx` (routing), `lib/api.ts` (HTTP client)

**`src/pages/`:**
- Purpose: Full-page React components (routed by `App.tsx`)
- Contains: `TicketList.tsx`, `TicketDetail.tsx`, `TicketForm.tsx`, `Login.tsx`, `UserList.tsx`, `Reports.tsx`, `Settings.tsx`, `KnowledgeBase.tsx`, etc.
- Naming: `PascalCase.tsx` for page components
- Pattern: Each page imports domain hooks and UI components, manages page-level state

**`src/components/`:**
- Purpose: Reusable UI components
- Contains: `TicketTable.tsx`, `KanbanView.tsx`, `TicketCard.tsx`, `FileUpload.tsx`, `SearchBar.tsx`, domain-specific: `TicketChecklist.tsx`, `TicketComments.tsx`, etc.
- Naming: `PascalCase.tsx`
- Pattern: ~50 components; mostly stateless presentation with hooks for data; `ui/` subdirectory contains shadcn imports

**`src/lib/`:**
- Purpose: Utilities and helper functions
- Contains:
  - `api.ts`: ApiClient class for all HTTP requests
  - `validations.ts`: Zod schemas for form validation
  - `tokenRefresh.ts`: Axios interceptor for automatic token refresh
  - `appearance.ts`: Theme (light/dark) + font persistence
  - `secureFileAccess.ts`: Secure attachment download helper
  - `storage.ts`: localStorage wrapper
  - `utils.ts`: General utilities

**`src/hooks/`:**
- Purpose: Custom React hooks (mostly data-fetching)
- Contains: `useTickets()`, `useUsers()`, `useCategories()`, `useTags()`, `useTemplates()`, domain-specific: `useTicketAttachments()`, `useTicketComments()`, etc.
- Naming: `useXxx.ts`
- Pattern: Each wraps React Query for server state; includes mutation functions for CRUD

**`src/contexts/`:**
- Purpose: React context providers
- Contains: `AuthContext.tsx` (authentication state)
- Pattern: One context per cross-cutting concern (currently auth only)

**`src/types/`:**
- Purpose: Shared TypeScript type definitions
- Contains: `ticket.ts` (Ticket, Comment, Checklist, Tag, Category), `filterView.ts`
- Pattern: Interfaces matching database schema; used by frontend + API client

**`server/src/`:**
- Purpose: Backend application source code
- Contains: Express app, routes, database, middleware, config, utilities

**`server/src/routes/`:**
- Purpose: Express route handlers (REST API endpoints)
- Contains: One file per domain (tickets, auth, categories, attachments, kb, etc.)
- Naming: `domain.ts` (e.g., `tickets.ts`)
- Pattern: Each exports `Router` instance with route definitions; uses `authenticate` middleware for protected endpoints

**`server/src/db/`:**
- Purpose: Database management and migrations
- Contains:
  - `connection.ts`: SQLite instance, schema initialization, column existence checks
  - `schema.sql`: Complete SQL schema (tables, indexes, constraints)
  - `init.ts`: Entry point for database initialization
  - Migration scripts: `add-refresh-tokens.ts`, `add-ticket-reminders.ts`, etc.
- Pattern: Each migration is idempotent (checks before altering)

**`server/src/lib/`:**
- Purpose: Business logic and utility libraries
- Contains:
  - `email.ts`: Nodemailer config, email templates for notifications
  - `reminderScheduler.ts`: Node-cron job that runs reminders
  - `autoCloseScheduler.ts`: Automatically closes old resolved tickets
  - `automationHelper.ts`: Auto-tagging and priority detection logic
- Pattern: Pure functions or classes; called by routes and scheduled tasks

**`server/src/config/`:**
- Purpose: Application configuration
- Contains: `passport.ts` (authentication strategies)

**`server/src/middleware/`:**
- Purpose: Express middleware functions
- Contains: `auth.ts` (JWT verification), `rateLimit.ts` (rate limiting)
- Pattern: Middleware functions that attach to app or route-level

## Naming Conventions

**Files:**
- React components: `PascalCase.tsx` (e.g., `TicketList.tsx`, `FileUpload.tsx`)
- Utilities/helpers: `camelCase.ts` (e.g., `api.ts`, `validations.ts`, `tokenRefresh.ts`)
- Routes: `camelCase.ts` or domain name (e.g., `tickets.ts`, `auth.ts`)
- Types/Interfaces: `PascalCase` (e.g., `Ticket`, `Comment`, `User`)

**Directories:**
- Lowercase plural for domain groupings: `pages/`, `components/`, `hooks/`, `routes/`
- Lowercase: `lib/`, `config/`, `middleware/`, `db/`, `types/`, `contexts/`
- Special: `ui/` for shadcn generated components

**Functions:**
- React components: `PascalCase` (e.g., `TicketList`, `FileUpload`)
- React hooks: `useXxx` (e.g., `useTickets`, `useCategories`)
- Utility functions: `camelCase` (e.g., `escapeCSVField()`, `generateCSV()`)
- Query keys: `xxxKeys` (e.g., `ticketKeys`, `userKeys`)

**Constants:**
- UPPERCASE_SNAKE_CASE (e.g., `UPLOAD_DIR`, `DB_PATH`, `API_BASE_URL`)
- Domain constants (lookups): lowercase (e.g., `statusLabels`, `priorityMap`)

**Type Names:**
- Domain types: `PascalCase` (e.g., `Ticket`, `User`, `Category`)
- Type suffixes: `-Input` (form input), `-Response` (API response)

## Where to Add New Code

**New Feature - Typical Pattern:**

1. **API Endpoint:**
   - Add route handler in `server/src/routes/[domain].ts`
   - Example: POST `/api/tickets/:id/assign` ‚Üí `assignTicketToUser()` function

2. **Database Schema:**
   - Modify `server/src/db/schema.sql` if new table/column needed
   - Create migration in `server/src/db/[feature-name].ts` (idempotent)
   - Call migration in `server/src/db/connection.ts` ‚Üí `initializeDatabase()`

3. **Frontend Hook:**
   - Create `src/hooks/use[Feature].ts` wrapping React Query
   - Define query keys: `export const [feature]Keys = { all: [...], ... }`
   - Example: `useAssignTicket()` mutation hook

4. **Frontend Component:**
   - Create `src/components/[Feature].tsx` if reusable
   - Otherwise integrate directly in page component
   - Import hook: `const { mutate } = useAssignTicket()`

5. **Page Integration:**
   - Add route in `src/App.tsx`: `<Route path="/feature" element={<FeaturePage />} />`
   - Create `src/pages/[Feature]Page.tsx` with layout

**New Component:**
- Path: `src/components/[ComponentName].tsx`
- Imports: Data from hooks, UI elements from `@/components/ui/`
- Pattern: Stateless presentation; hooks for async data

**New Utility:**
- Path: `src/lib/[utilityName].ts` (frontend) or `server/src/lib/[utilityName].ts` (backend)
- Export: Named exports for functions
- Pattern: Pure functions or classes

**Scheduled Job:**
- Create `server/src/lib/[jobName]Scheduler.ts`
- Implement: Async function that runs periodic tasks
- Register: Call in `server/src/index.ts` ‚Üí `start[Job]Scheduler()`
- Timing: Defined with node-cron syntax

- Generated: Yes (created on first run)

- Generated: Yes (by `npm run build`)
- Committed: No (in `.gitignore`)
